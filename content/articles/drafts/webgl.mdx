**WebGL (Web Graphics Library)** — это программный интерфейс, который позволяет отображать интерактивную 2D и 3D графику прямо в браузере без использования плагинов.

Если вы когда-то видели хотя бы одну красивую анимацию внутри элемента `<canvas>`, которую нельзя повторить средствами HTML и CSS,  можете не сомневаться - данная анимация сделана на WebGL.

По сути WebGL просто предоставляет нам API для отрисовки точек, линий и треугольников, а все остальное - ложится на наши плечи.

# Шейдеры
В каждой из программ WebGL, должны быть две функции, одна для вершинного шейдера и другая для фрагментного шейдера.

**Вершинный шейдер** — это программная функция, которая вычисляет положение каждой вершины на экране. Она получает атрибуты вершин (такие как координаты, цвет, текстурные координаты) в качестве входных данных и преобразует их в координаты экрана. Вершинный шейдер выполняется для каждой вершины геометрии и является первым этапом в процессе рендеринга WebGL.

**Фрагментный шейдер** — это программная функция, которая определяет цвет каждого пикселя (фрагмента) примитива. Она вызывается для каждого пикселя, который должен быть отрисован на экране. Фрагментный шейдер получает интерполированные данные от вершинного шейдера и может использовать различные техники для определения итогового цвета пикселя, такие как текстурирование, освещение, смешивание цветов и другие эффекты.

Звучит сложно... Если говорить по простому, то: вершинный шейдер отвечает за то, где будет находиться каждая точка на экране, а фрагментный шейдер определяет, каким цветом эта точка будет окрашена. Все остальное - просто дополнительные возможности и эффекты.
